# Law UI Components — 법령 분석 UI 구현 스킬

비법률 전공자가 IT 법령을 쉽게 이해할 수 있도록 설계된 UI 컴포넌트 구현 가이드. 정부 사이트의 정적이고 텍스트 중심적인 UX를 탈피하여 모던하고 직관적인 인터페이스를 구축한다.

## 디자인 원칙

1. **Progressive Disclosure** — 대시보드(전체 법령 요약) → 법령 상세(계층뷰) → 조문 상세(본문+해설+판례)
2. **카드 기반 레이아웃** — 넉넉한 여백, 그림자로 깊이감, 정보 단위별 카드
3. **시각적 계층** — 법률(파랑) → 시행령(초록) → 시행규칙(주황) → 고시(보라) 색상 코딩
4. **법률 용어 접근성** — 전문 용어에 호버 툴팁, AI 해설 토글 패널

## 색상 시스템

```
법률 계층 색상:
  법률:     blue-600 (#2563EB) / blue-50 배경
  시행령:   emerald-600 (#059669) / emerald-50 배경
  시행규칙: amber-600 (#D97706) / amber-50 배경
  고시:     purple-600 (#9333EA) / purple-50 배경

상태 색상:
  개정 예정:  amber-500 (경고)
  최근 개정:  red-500 (주의)
  시행 중:    green-500 (안정)

배경: slate-50 (밝은 모드), slate-900 (다크 모드)
텍스트: slate-900 / slate-100
```

## 핵심 컴포넌트 패턴

### 1. 법령 계층 그래프 (React Flow)

법률 → 시행령 → 시행규칙 → 고시 관계를 인터랙티브 노드 그래프로 표현.

```
노드 구조:
- 루트: 법률 노드 (파랑, 가장 큰 크기)
- 2단: 시행령 노드들 (초록)
- 3단: 시행규칙 노드들 (주황)
- 4단: 관련 고시 노드들 (보라)

엣지: 위임 관계를 화살표로 표현
클릭: 노드 클릭 시 해당 법령 상세로 이동
```

데이터 소스: `chain_law_system` → `get_three_tier` → `get_law_tree`

### 2. 3단 위임조문 비교 뷰

법률 조문이 시행령/시행규칙의 어떤 조문에 위임하는지 3단 컬럼으로 시각화.

```
┌──────────────┬──────────────┬──────────────┐
│   법률 조문   │  시행령 조문  │ 시행규칙 조문 │
├──────────────┼──────────────┼──────────────┤
│ 제6조 (인증)  │ 제7조 (인증  │ 제3조 (인증  │
│ ...대통령령   │  기준)       │  절차)       │
│ 으로 정한다   │              │              │
└──────────────┴──────────────┴──────────────┘
```

위임 관계가 있는 행은 연결선으로 하이라이트.

### 3. 조문 탐색기

좌측 목차 + 우측 본문 레이아웃.

```
┌─────────┬─────────────────────────────┐
│ 목차     │ 제2조 (정의)                │
│         │                             │
│ 제1장   │ ① "개인정보"란 살아 있는    │
│  제1조  │ 자연인에 관한 정보...         │
│  제2조 ←│                             │
│  제3조  │ [쉬운 해설 토글]             │
│ 제2장   │ ┌──────────────────────┐    │
│  ...    │ │ 💡 쉬운 해설          │    │
│         │ │ 이름, 주민번호 등      │    │
│         │ │ 개인을 알아볼 수       │    │
│         │ │ 있는 정보를 말합니다   │    │
│         │ └──────────────────────┘    │
│         │                             │
│         │ 📋 관련 판례 (3건)           │
│         │ ┌────────────────────┐      │
│         │ │ 대법원 2023다12345  │      │
│         │ │ [요약 보기]         │      │
│         │ └────────────────────┘      │
└─────────┴─────────────────────────────┘
```

### 4. 신구법 비교 뷰어 (diff2html)

side-by-side 또는 inline 모드 전환 가능한 diff 뷰어.

```
모드 전환: [Side-by-side] [Inline]

┌──────────────────┬──────────────────┐
│ 개정 전           │ 개정 후           │
├──────────────────┼──────────────────┤
│ 제5조 ①항         │ 제5조 ①항         │
│ -정보통신서비스    │ +정보통신서비스    │
│  제공자는...       │  제공자 등은...    │
│                   │ +② (신설)         │
│                   │  개인정보처리자는  │
└──────────────────┴──────────────────┘
```

삭제: 빨간 배경, 추가: 초록 배경. diff2html 라이브러리 활용.

### 5. AI 검색바

```
┌──────────────────────────────────────┐
│ 🔍 클라우드 보안 인증 기준이 뭐야?    │
├──────────────────────────────────────┤
│ 필터: [법률 ▼] [시행령 ▼] [고시 ▼]  │
├──────────────────────────────────────┤
│ 결과:                                │
│ ┌────────────────────────────────┐  │
│ │ 클라우드발전법 제23조 (인증)    │  │
│ │ 관련도: 95% | 시행: 2024.01.01 │  │
│ │ "클라우드컴퓨팅서비스의 품질... │  │
│ └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 6. 개정 타임라인

수직 타임라인으로 개정 이력 시각화.

```
──●── 2024.03.15 전부개정 (시행: 2024.09.15)
  │   ⚠️ 시행 D-90 알림 대상
  │
──●── 2023.09.20 일부개정
  │   제5조, 제12조 변경
  │   [신구대조 보기]
  │
──●── 2022.01.01 최초 시행
```

### 7. 문서 분석 결과

리스크 레벨별 색상 하이라이트.

```
위험도: 🔴 높음 | 🟡 보통 | 🟢 낮음

┌────────────────────────────────────┐
│ 제3조 (개인정보 수집)        🔴    │
│ "사전 동의 없이 수집 가능"         │
│ → 개인정보보호법 제15조 위반 가능  │
├────────────────────────────────────┤
│ 제7조 (데이터 보관)          🟡    │
│ "무기한 보관"                      │
│ → 정보통신망법 제29조 확인 필요    │
└────────────────────────────────────┘
```

## 반응형 전략

- **Desktop**: 3단 컬럼 (목차 + 본문 + 사이드패널)
- **Tablet**: 2단 → 사이드패널은 슬라이드 오버
- **Mobile**: 1단 스택 + 하단 네비게이션, 계층 그래프는 수직 리스트로 대체
